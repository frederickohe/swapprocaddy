Swappro VPS deploy flow
=======================

Server layout (/var/www):
  swapbackend/
  swapprocaddy/    ← run setup scripts from here
  swapsite/

Prerequisites
-------------
- Docker + Docker Compose installed
- DNS A records for all hosts → VPS public IP:
    swappro.store, www, api, db, portainer
- Ports 80 and 443 open on the firewall

One-time setup
--------------
cd /var/www/swapprocaddy
chmod +x setup-networks.sh setup-ssl.sh
./setup-networks.sh

Deploy / update stacks
----------------------
cd /var/www/swapbackend
git pull
docker compose down
docker compose up --build -d

cd /var/www/swapprocaddy
git pull
docker compose up -d

Issue / refresh TLS certificates
----------------------------------
cd /var/www/swapprocaddy
./setup-ssl.sh

Full first-time deploy (after git pull in both repos)
-----------------------------------------------------
cd /var/www/swapprocaddy
./setup-networks.sh
cd ../swapbackend && docker compose up --build -d
cd ../swapprocaddy && docker compose up -d
./setup-ssl.sh

Verify
------
https://www.swappro.store
https://api.swappro.store
https://db.swappro.store
https://portainer.swappro.store

CI/CD (GitHub Actions)
----------------------
Each repo deploys on push to `main` via `.github/workflows/deploy.yml`.

Server paths and scripts:
  /var/www/swapsite/scripts/deploy.sh      → git pull + restart Caddy
  /var/www/swapbackend/scripts/deploy.sh   → git pull + docker compose up --build -d
  /var/www/swapprocaddy/scripts/deploy.sh  → git pull + docker compose up -d

One-time GitHub secrets (all three repos):
  DEPLOY_HOST     = 13.140.185.135
  DEPLOY_USER     = deploy
  DEPLOY_SSH_KEY  = private key from ~/.swappro-deploy/github_actions_deploy

After `gh auth login`, run from swapprocaddy:
  powershell -ExecutionPolicy Bypass -File scripts/setup-github-secrets.ps1

Manual deploy on server:
  bash /var/www/<repo>/scripts/deploy.sh
